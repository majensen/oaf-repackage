OAF Manual

  • Name

    oaf.pl - Ornithine decarboxylase Antizyme Finder

  • Synopsis

    oaf.pl [options] mode

  • Description

    Ornithine decarboxylase Antizyme Finder (OAF) is a specialized software tool for identification of genes encoding antizyme and for annotating the sites of programmed ribosomal frameshifting utilized in antizyme genes expression.
    oaf.pl is an implementation the oaf.pm bioperl module. oaf.pl takes a sequence from an input stream and finds an eukaryotic ornithine decarboxylase antizyme. As input, it takes nucleotide or protein sequences in FASTA format or Genbank accession numbers. HMM profiles are used in order to identify location, frame and orientation of such gene.

  • Modes

    --sequence=<sequence file>
    Use a FASTA file (with one or more DNA/RNA sequences) as input.
    --genbank=<accession>
    Enable a search in GenBank. Provide an accession number rather than a sequence file.
    --ortho=<aa sequences file>
    Start by searching for similar protein sequence(s) (FASTA file format) in a database, and use those as queries for OAF.

  • Options

    -a
    Force the use of alternative start codons, according to the current genetic code table. Otherwise, ATG is the only allowed initiation codon.
    --expect
    Set the E-value threshold. This setting specifies the statistical significance threshold for reporting matches against database sequences. The default value (1e-40) means that 1e-40 such match is expected to be found merely by chance, according to the stochastic model of Karlin and Altschul (1990). If the statistical significance ascribed to a match is greater than the EXPECT threshold, the match will not be reported. Lower EXPECT thresholds are more stringent, leading to fewer chance matches being reported.
    --format
    Available output format codes include 'fasta' (FASTA format); 'genbank' (Genbank format); 'raw' (raw sequence, no other information); 'xml' (XML format). By default the FASTA format is used. The Document Type Definition (DTD) for the XML format is available here.
    --name
    Overwrite the sequence name by the provided one. Otherwise the program will use the sequence name from as input.
    --table
    Specify a genetic code table used for the translation of the query. By default, for the genbank and ortho modes, the genetic code table is used according to genbank annotation. The sequence mode uses the Standard Code (1). Available tables, see http://www.ncbi.nlm.nih.gov/Taxonomy/Utils/wprintgc.cgi?mode=c

  • Expert Options

    --bank
    Set the database used by the ortho[log] mode. For the remote BLAST at NCBI (when -r is set) the available databases are either 'refseq_rna' (correspondes to NCBI Reference mRNA sequences database) or 'est' (sequences from EST divisions). By default 'est' is used.
    -d
    Turn off the pre-filter for long sequences and use user-provided sequence directly for OAZ search. Normally, the sequences of more than 20kb are pre-filtered by a local fasta search in order to minimize the number of potential OAZ encoding sequences. This option will usually increase the sensitivity of the search, if the OAZ is VERY divergent, but that will dramatically increase the time and memory used (so don't do it without good reason!)
    --filter
    Filter the sequences shown. Normally OAF.pl show all sequences identified icluding putative/ambiguous which correspond to ESTs containing truncated CDS, sequence errors or a missing +1 frameshift. By default there is no filter (0), you can filter for the non-ambiguous hits only (1) or the ambiguous hits only (2).
    -r
    Configure the program to use the remote BLAST server of the NCBI rather than with a local BLAST.
    -v
    Print more possibly useful stuff, such as the individual scores for each sequence.

  • Examples

    ./oaf.pl --sequence=example.fasta
    Simple request, use an FASTA format file (with one or more DNA/RNA sequences) as request.
    ./oaf.pl -a --table=6 --name=my_sequence --sequence=example.fasta
    Do the same search, but force the Ciliate nuclear genetic code to be used. Allow the alternative start codon and rename the output sequence to 'my_sequence'
    ./oaf.pl --format=xml --genbank=NM_004152
    Retrieve the sequence NM_004152 from GenBank, find an OAZ gene and show an XML report.
    ./oaf.pl --orhto=example.aa
    Use the protein sequences (FASTA format) to retrieve putative ortholog sequence and use them as query.
    ./oaf.pl --expect=2e-5 -r --bank=refseq_rna --ortho=example.aa
    Use the NCBI remote blast server, and the 'refseq_rna' database to retrieve the putative ortholog sequences. Set the expected value (E-value) at 2e-5.
